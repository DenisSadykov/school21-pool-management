# 👨‍💼 Гайд для администраторов

Инструкция для администраторов системы управления бассейном.

## 🎯 Основные обязанности

1. **Следить за работой системы** — проверять логи, статус
2. **Управлять пользователями** — добавлять/удалять волонтёров
3. **Синхронизировать данные** — обновлять Google Sheets
4. **Резервные копии** — регулярно скачивать данные
5. **Помощь Team Leads** — ответ на вопросы

## 🔐 Права доступа

### Роли в системе:
- **Admin** (администратор) — полный доступ ко всему
- **Team Lead** (тимлид) — управление сменами и волонтёрами
- **Volunteer** (волонтёр) — только просмотр и выбор смен

## 📊 Панель администратора

Всё управление находится в разделе "⚙️ Админ" (для Admin только).

### Основные действия:

#### 🔄 Синхронизация с Google Sheets
```
Панель Админ → Синхронизация данных → Синхронизировать
```
- Загружает/скачивает данные из Google Sheets
- Должна происходить минимум раз в день
- Можно настроить автоматическую синхронизацию

#### ⬇️ Экспорт данных
```
Панель Админ → Экспорт данных → Экспортировать
```
- Скачивает все данные в JSON
- Используется для резервной копии
- Требуется перед обновлением системы

#### 🗑️ Сброс базы данных
```
Панель Админ → Опасные операции → Сбросить базу данных
```
⚠️ **Осторожно!** Удалит ВСЕ данные. Используйте только:
- В тестовых целях
- Если данные полностью повреждены
- После создания резервной копии

## 📋 Управление волонтёрами

### Добавить волонтёра

1. Перейти в "👥 Волонтёры"
2. Волонтёры добавляются автоматически при входе в систему
3. Админ может просмотреть статистику каждого

### Блокировка волонтёра

Если волонтёр нарушает правила:
1. Открыть его профиль
2. Нажать "Заблокировать"
3. Волонтёр не сможет выбирать смены

## 🔍 Мониторинг системы

### Ежедневная проверка:

```
09:00 - Проверить, все ли смены созданы
11:00 - Синхронизировать с Google Sheets
15:00 - Проверить новых волонтёров
18:00 - Синхронизировать перед ночью
22:00 - Финальная проверка
```

### Что смотреть:

- **Таблица смен** — есть ли необходимые смены?
- **Волонтёры** — активны ли все участники?
- **Награды** — корректно ли считаются коины?
- **Логи ошибок** — нет ли проблем в системе?

## 🔧 Техническое обслуживание

### Еженедельно:

1. **Резервная копия**
```bash
# В админ-панели
Экспорт данных → Скачать → Сохранить на диск
```

2. **Проверка производительности**
   - Открыть Админ-панель
   - Посмотреть статистику сервера
   - Всё ли работает быстро?

3. **Проверка логов**
```bash
# На сервере
tail -f backend/logs/app.log
```

### Ежемесячно:

1. **Глубокая очистка данных**
   - Удалить старые/неиспользуемые записи
   - Архивировать замыслил смены

2. **Обновление системы**
```bash
git pull origin main
pip install -r backend/requirements.txt
npm update --prefix frontend
```

3. **Тестирование всех функций**
   - Создать тестовую смену
   - Добавить волонтёра
   - Проверить синхронизацию

## 📞 Проблемы и решения

### Система медленная

**Причины:**
- Много данных в БД
- Сервер слабый
- Сетевые проблемы

**Решение:**
- Архивировать старые данные
- Увеличить мощность сервера
- Оптимизировать запросы

### Синхронизация не работает

**Причины:**
- Google API ключ истёк
- Нет интернета
- Лимит API превышен

**Решение:**
```bash
# Проверить логи
tail -f backend/logs/sync.log

# Пересоздать ключ API
# Смотрите docs/google-sheets-sync.md
```

### Волонтёр не может войти

**Причины:**
- Заблокирован администратором
- Сезон закончился
- Технические проблемы

**Решение:**
- Проверить статус в системе
- Разблокировать если необходимо
- Перезагрузить браузер

## 🚨 Аварийные процедуры

### Система полностью упала

```bash
# 1. Проверить статус
docker ps  # Все ли контейнеры запущены?

# 2. Перезагрузить
docker-compose restart

# 3. Если не помогло
docker-compose down
docker-compose up

# 4. Проверить логи
docker-compose logs -f backend
```

### Потеря данных

```bash
# 1. Проверить резервную копию
ls -la backup_*.json

# 2. Восстановить из JSON
python scripts/restore_backup.py backup_2026-06-22.json

# 3. Синхронизировать
curl -X POST http://localhost:5000/api/sync
```
